Tipos de datos y propiedades de campo en Access

Cuando se diseña y crea una base de datos, tenemos que planificar una o más tablas así como los campos (columnas) para cada tabla (recordemos que todas las tablas de Access se componen de campos), y ademas es necesario elegir un tipo de datos para cada campo.

Tipos de datos en Access

El tipo de datos de un campo es la propiedad más importante porque determina qué tipo de datos puede almacenar el campo, como puede ser una gran cantidad de texto o archivos adjuntos. Luego, una vez elegido el tipo de datos adecuado a la informacion que almacenaremos en dicho campo. Las propiedades de un campo describen las características y el comportamiento de los datos agregados a dicho campo.

Cada campo tiene un tipo de datos. El tipo de datos de un campo indica el tipo de datos que almacena el campo, como una gran cantidad de texto o archivos adjuntos.

Por ejemplo, si necesita almacenar fechas y horas, establezca un campo cuyo tipo de dato sea Fecha y Hora. Si necesita almacenar nombres y direcciones, establezca uno o varios campos en el tipo de datos Texto y así respectivamente. Access ofrece muchos tipos de datos diferentes y cada tipo tiene una finalidad concreta.

Por último, los tipos de datos proporcionan una forma básica de validación de datos porque ayudan a garantizar que los usuarios escriben los tipos de datos adecuados en los campos de tabla. Por ejemplo, no se puede escribir texto en un campo configurado para aceptar solo números.

El tipo de datos de un campo determina multiples cualidades importantes del mismo, como las siguientes:

  1. Los formatos que se pueden usar con el campo.
  2. El tamaño máximo de un valor del campo.
  3. Cómo se puede usar el campo en expresiones.
  4. Si el campo se puede indizar.

El tipo de datos Texto permite escribir/almacenar casi cualquier carácter (letra, símbolo o número). Pero un campo cuyo tipo de datos es número solo puede almacenar datos numéricos. Sin embargo, una cuidada selección de tipos de datos puede ayudarle a aprovecharse de características adicionales de Access (como la validación de datos y las funciones) y a mejorar la precisión de la información que está almacenando.

Propiedades de campo: Permitir longitud cero y requerido

Cuando creamos nuevas tablas en Access, es mucho mas fácil realizarlo a través de la vista diseño de la tabla. Para modificar y/o agregar información también podemos acceder a dicha vista.
  1. Permitir longitud cero
    2.

    Permitir longitud cero admite dos posibles valores, Sí y No. Si está con valor Sí significa que podemos introducir una cadena de longitud cero en el campo. Esta cadena se puede puede representar como "" sin nada dentro de las comillas. Con el valor No no se admitiría esta cadena.

  2. Requerido
    3.

    La propiedad Requerido indica que ese campo no se puede dejar en blanco, sin datos.

    Observación: Todos los campos pertenecientes a la clave principal de una tabla tienen la obligación de poseer datos, no pueden estar vacíos.

    El resto de campos no tienen, en principio, porque ser requerido. Pero si la propiedad Requerido de un campo está a Sí, significa que no podemos dejar de introducir datos en ese campo. Necesariamente introduciremos información para cumplir con la exigencia de esta propiedad.

Sin embargo, puede ocurrir que, en ciertas ocasiones, no tenemos el dato para ese campo y resulta que con Requerido activado, Access nos obliga a introducir información en el mismo. ¿Qué podemos hacer? Si la propiedad permitir longitud cero está activada (valor Sí), que hacemos es pulsar la barra espaciadora, y de esta forma podemos pasar a otro campo sin problemas. Más tarde, con la información en nuestras manos, introducimos el dato sin problemas.

En resumen: Con permitir longitud cero y requerido a sí podemos dejar la introducción de datos para más tarde. Si tuviésemos requerido a si y permitir longitud cero a no, entonces Access no permite introducir una cadena de longitud cero en el campo, y estaremos obligados a introducir información, dar a la barra espaciadora no se permite.

Propiedades de campo: Mascara de entrada

Se puede usar una máscara de entrada para validar los datos, lo que obliga a los usuarios a escribir los valores de una manera específica. Por ejemplo, una máscara de entrada permite obligar a los usuarios a escribir las fechas en un formato como 14/04/2007.

Una máscara de entrada está formada por caracteres literales (como espacios, puntos, guiones y paréntesis) que separan los espacios en blanco para rellenar. La configuración de la propiedad Máscara de entrada (InputMask) está formada por caracteres literales junto con caracteres especiales que determinan el tipo de valor que puede introducirse en un espacio en blanco en esa posición. Una buena pratica es configurar máscaras de entrada para los campos que contienen datos cuyo formato siempre es el mismo, permite que los usuarios escriban datos correctamente en la base de datos de Access.

Observacion: Una máscara de entrada solo determina si Access acepta los datos o no; es decir, no altera la manera en la que los datos se almacenan, que se controla mediante el tipo de datos del campo y otras propiedades.

Las máscaras de entrada se utilizan principalmente en campos de Texto y de Fecha/Hora, pero también pueden utilizarse en campos Numéricos o de Moneda. Por ejemplo, podemos usar una máscara de entrada para asegurarnos que los números de teléfono introducidos en un campo de número de teléfono van a tener el formato adecuado.

Las máscaras de entrada están compuestas de una sección obligatoria y dos secciones opcionales, cada una de ellas se separa con punto y coma. El propósito de cada sección es el siguiente:

  • La primera sección es obligatoria. Incluye los caracteres o la cadena de caracteres que especifican propiamente a la mascara de entrada junto con marcadores de posición tales como como paréntesis, puntos y guiones. Para ver una lista de los caracteres que puedes utilizar para definir la máscara de entrada, mira la tabla que aparece a continuación.
  • La segunda sección es opcional y hace referencia a los caracteres de máscara insertados y al modo en que se almacenan dentro del campo, esto es, en esta seccion se especifica si Access 2016 almacena los caracteres de visualización literales en la tabla al introducir datos. Si utilizas 0 para esta sección, todos los caracteres de visualización literales (por ejemplo, el paréntesis de una máscara de entrada de un número de teléfono) se almacenan con el valor; si introduces 1 o dejas esta sección en blanco, sólo se almacenan los caracteres introducidos.
  • La tercera sección también es opcional e indica un solo carácter o espacio que se usa como marcador de posición. De forma predeterminada, Access usa el carácter de subrayado (_). Si desea usar otro carácter, escríbalo en la tercera sección de la máscara.
