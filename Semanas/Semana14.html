Microsoft Access

Hoy en día muchas de las actividades que se desempeñan en una empresa requieren un seguimiento especial. Algunas de esas tareas son:

  • Administrar un inventario,
  • Seguimiento de pedidos realizados por los clientes,
  • Control de ingresos y gastos,

Aunque podríamos desempeñar esta tarea en Excel. Nada es más cómodo que utilizar una base de datos, y una opción para esto último es Microsoft Access. Con Access, puedes crear una base de datos sin escribir código ni ser un experto en bases de datos. La diferencia mas grande entre Excel y Access es la forma en la que Access permite relaciones entre múltiples tablas.

Microsoft Access (MS Access) es un Sistema Gestor de Bases de Datos Relacional; esto significa que está preparado para almacenar múltiples tablas y crear relaciones entre estas. Fue desarrollado por Microsoft y permite crear bases de datos relacionales. La información se obtiene a través de tablas, las cuáles almacenan información sobre temas concretos. Las tablas disponen los datos en campos (columnas) y registros (filas).

Una base de datos en Access nos permite generar nueva información a partir de dichos datos y a través de diversos procesos, haciendo uso de campos y registros. Las bases de datos de Access 2010 tienen la extensión .accdb o en versiones mas recientes la extensión .mdb para que el ordenador las reconozca como tal.

Access es una poderosa herramienta que sirve como gestor de datos, ademas de que se puede conectar con Excel y todos los otros programas de Officce.

Tipos de datos en Access

Cuando se diseña y crea una base de datos, tenemos que planificar una o más tablas así como los campos (columnas) para cada tabla (recordemos que todas las tablas de Access se componen de campos), y ademas es necesario elegir un tipo de datos para cada campo.

El tipo de datos de un campo es la propiedad más importante porque determina qué tipo de datos puede almacenar el campo, como puede ser una gran cantidad de texto o archivos adjuntos.

Ejemplo

  • Si necesitas almacenar fechas y horas, establece un campo cuyo tipo de dato sea Fecha y Hora.
  • Si necesita almacenar nombres y direcciones, establezca uno o varios campos en el tipo de datos Texto y así respectivamente.

    • Access ofrece muchos tipos de datos diferentes y cada tipo tiene una finalidad concreta.

Observación

Los tipos de datos proporcionan una forma básica de validación de datos porque ayudan a garantizar que los usuarios escriben los tipos de datos adecuados en los campos de tabla. Por ejemplo, no se puede escribir texto en un campo configurado para aceptar solo números.

El tipo de datos de un campo determina multiples cualidades importantes del mismo, como las siguientes:

  1. Los formatos que se pueden usar con el campo.
  2. El tamaño máximo de un valor del campo.
  3. Cómo se puede usar el campo en expresiones.
  4. Si el campo se puede indizar.

El tipo de datos Texto permite escribir/almacenar casi cualquier carácter (letra, símbolo o número). Pero un campo cuyo tipo de datos es número solo puede almacenar datos numéricos. Sin embargo, una cuidada selección de tipos de datos puede ayudarle a aprovecharse de características adicionales de Access (como la validación de datos y las funciones) y a mejorar la precisión de la información que está almacenando.

Tipo de datos Uso Tamaño
Texto corto Datos alfanuméricos cortos como nombres, apellidos, títulos, una dirección, etc. En general se utiliza para introducir datos que incluyan texto y/o valores numéricos que no requieran cálculos matemáticos. Hasta 255 caracteres.
Texto largo Grandes cantidades de datos alfanuméricos: oraciones y párrafos. Un uso típico de un campo Texto Largo sería una descripción detallada de un producto. Hasta 1 gigabyte (GB) aproximadamente, pero los controles para mostrar un texto largo se limitan a los primeros 64 000 caracteres.
Número Datos numéricos que vayan a ser utilizados para realizar cálculos matemáticos.

1, 2, 4, 8 o 16 bytes.

Datos adjuntos

Se usa en un campo que permita adjuntar archivos o imágenes a un registro. Por ejemplo, si tiene una base de datos de contactos de trabajo, puede usar un campo de datos adjuntos para adjuntar una foto del contacto, o adjuntar documentos como un currículo.

A diferencia del tipo Objeto OLE, cada documento, gráfico, hoja de cálculo, etc que se adjunte a un campo de este tipo se ejecutará en el programa que fue creado, sin requerir más espacio adicional en la base de datos del que necesita el solo enlace. Admite varios archivos.

El tipo de datos Datos adjuntos no está disponible en los formatos de archivo MDB.

Luego, una vez elegido el tipo de datos adecuado a la informacion que almacenaremos en dicho campo. Las propiedades de un campo describen las características y el comportamiento de los datos agregados a dicho campo.

Propiedades de un campo en Access

Después de crear un campo y establecer su tipo de datos, se pueden establecer otras propiedades del campo. Por ejemplo, se puede controlar el tamaño de un campo de texto estableciendo su propiedad Tamaño del campo, entre otras muchas propiedades.

Enseguida se describen algunas propiedades de campo y su uso para distintos tipos de datos:

Propiedad Uso
Formato Determina la forma en que aparece el campo cuando se visualiza o imprime en hojas de datos o en formularios o informes enlazados al campo. Se puede definir un formato personalizado para un campo de hipervínculo.
Si/No Opciones de Sí o No.
Titulo Se utiliza esta propiedad para indicar cómo queremos que se visualice la cabecera del campo. Por ejemplo, si un campo se llama Fecha_nac e indicamos Fecha de nacimiento como valor en la propiedad Título, en la cabecera del campo Fecha_nac veremos Fecha de nacimiento. Es tambien el el texto de etiqueta que se muestra para este campo de forma predeterminada en formularios, informes y consultas. Si esta propiedad está vacía, se usa como valor predeterminado el nombre del campo. Se permite cualquier cadena de texto. Un título eficaz suele ser breve.
Valor predeterminado El valor predeterminado es el valor que se almacenará automáticamente en el campo si no introducimos ningún valor. Se suele emplear cuando se sabe que un determinado campo va a tener la mayoría de las veces el mismo valor, se utiliza esta propiedad para indicar o especificar cuál va a ser ese valor y así que se introduzca automáticamente en el campo a la hora de introducir los datos de la tabla.
Regla de validación Proporciona una expresión que debe ser verdadera al agregar o cambiar el valor en este campo. Se usa junto con la propiedad Texto de validación. Esta propiedad nos permite controlar la entrada de datos según el criterio que se especifique.
Texto de validación Escribir un mensaje que aparezca cuando un valor especificado infrinja la expresión en la propiedad Regla de validación.
Requerido Requiere que los datos se introduzcan en un campo.
Permitir longitud cero Permite escribir (al establecerla en Sí) una cadena de longitud cero ("") en un campo Hipervínculo, Texto o Memo.
Indexado

Especifica si el campo tiene un índice. Hay tres valores disponibles:

  1. Sí (sin duplicados) Crea un índice exclusivo en el campo.
  2. Sí (con duplicados) Crea un índice no exclusivo en el campo.
  3. No, quita cualquier índice del campo.

Nota: No cambie esta propiedad en un campo que se use como clave principal.
Compresión Unicode Comprime el texto que se almacena en este campo cuando se almacenan menos de 4.096 caracteres.
Modo IME Controla la conversión de caracteres en las versiones asiáticas de Windows.
Alinear texto Especifica la alineación de texto predeterminada dentro de un control.

Permitir longitud cero y requerido

Cuando creamos nuevas tablas en Access, es mucho mas fácil realizarlo a través de la vista diseño de la tabla. Para modificar y/o agregar información también podemos acceder a dicha vista.

  • Longitud cero:

    Permitir longitud cero admite dos posibles valores, Sí y No. Si está con valor Sí significa que podemos introducir una cadena de longitud cero en el campo. Esta cadena se puede puede representar como "" sin nada dentro de las comillas. Con el valor No no se admitiría esta cadena.

  • Requerido:

    La propiedad Requerido indica que ese campo no se puede dejar en blanco, sin datos.

Observación.

Todos los campos pertenecientes a la clave principal de una tabla tienen la obligación de poseer datos, no pueden estar vacíos. El resto de campos no tienen, en principio, porque ser requerido. Pero si la propiedad Requerido de un campo está a Sí, significa que no podemos dejar de introducir datos en ese campo. Necesariamente introduciremos información para cumplir con la exigencia de esta propiedad.

Sin embargo, puede ocurrir que, en ciertas ocasiones, no tenemos el dato para ese campo y resulta que con Requerido activado, Access nos obliga a introducir información en el mismo. ¿Qué podemos hacer? Si la propiedad permitir longitud cero está activada (valor Sí), que hacemos es pulsar la barra espaciadora, y de esta forma podemos pasar a otro campo sin problemas. Más tarde, con la información en nuestras manos, introducimos el dato sin problemas.

En resumen: Con permitir longitud cero y requerido a sí podemos dejar la introducción de datos para más tarde. Si tuviésemos requerido a si y permitir longitud cero a no, entonces Access no permite introducir una cadena de longitud cero en el campo, y estaremos obligados a introducir información, dar a la barra espaciadora no se permite.
